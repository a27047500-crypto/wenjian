# Flow Docs 云服务器部署

## 适用场景

- 5 到 50 人先在公司内部使用
- 一台 Linux 云服务器长期运行
- 域名或公网 IP 统一访问

## 推荐配置

- 2 vCPU
- 4 GB RAM
- 40 GB SSD
- Ubuntu 24.04 LTS

## 第一步：上传项目

把整个目录上传到服务器，例如：

`/root/sop-company-app`

## 第二步：安装基础环境

在服务器执行：

```bash
cd /root/sop-company-app
chmod +x deploy/linux/setup-ubuntu.sh deploy/linux/install.sh deploy/linux/update.sh deploy/linux/backup.sh
sudo ./deploy/linux/setup-ubuntu.sh
```

## 第三步：安装应用

先按你的实际域名设置 `DOMAIN`。如果先用公网 IP，可以直接保留 `_`。

```bash
cd /root/sop-company-app
sudo DOMAIN=_ PORT=3100 ./deploy/linux/install.sh
```

安装完成后，应用会被部署到：

- 程序目录：`/opt/sop-company-app`
- 数据目录：`/var/lib/sop-company-app/data`
- 环境变量：`/etc/sop-company-app.env`
- systemd 服务：`sop-company-app.service`

## 第四步：查看状态

```bash
systemctl status sop-company-app.service --no-pager
curl http://127.0.0.1:3100/api/health
```

## 第五步：开放访问

如果云厂商有安全组，需要放行：

- `80`
- `443`

如果服务器本机开了防火墙，也要放行：

```bash
sudo ufw allow 80
sudo ufw allow 443
```

## 第六步：后续更新

上传新版本后执行：

```bash
cd /root/sop-company-app
sudo ./deploy/linux/update.sh
```

## 第七步：备份数据

```bash
sudo ./deploy/linux/backup.sh
```

备份默认输出到：

`/var/backups/sop-company-app`

## 重要文件

- `deploy/linux/install.sh`
- `deploy/linux/update.sh`
- `deploy/linux/backup.sh`
- `deploy/linux/setup-ubuntu.sh`
- `.env.example`
