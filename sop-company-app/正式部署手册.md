# 流程文件管理平台正式部署手册

本文档按“买一台 Linux 云服务器，然后正式上线给公司使用”的方式编写。

## 一、建议的正式部署方案

推荐方案：

- 服务器系统：`Ubuntu 24.04 LTS`
- 部署方式：`Node.js + systemd + Nginx`
- 访问方式：`域名 + HTTPS`
- 部署目录：`/opt/sop-company-app`
- 数据目录：`/var/lib/sop-company-app/data`

适用规模：

- 20 到 50 人内部使用，建议 `2 核 4G` 起步
- 如果图片较多或后续并发更高，建议 `4 核 8G`

## 二、上线前准备

你需要先准备好 4 样东西：

1. 一台云服务器
2. 一个域名
3. 服务器公网 IP
4. 本项目完整目录

建议你先在云厂商控制台放通这些端口：

- `22`：用于 SSH 登录服务器
- `80`：用于 HTTP
- `443`：用于 HTTPS

如果你已经买好域名，需要提前把域名解析到服务器公网 IP：

- 比如把 `sop.yourcompany.com` 的 `A 记录` 指向你的服务器公网 IP

## 三、把项目上传到服务器

本地项目目录就是：

- [sop-company-app](</C:/Users/Administrator/Desktop/word/sop-company-app>)

你可以用 `WinSCP`、`FinalShell`、`Xftp` 或者 `scp` 上传。

推荐上传到服务器这个位置：

- `/root/upload/sop-company-app`

上传完成后，登录服务器：

```bash
ssh root@你的服务器公网IP
```

## 四、安装运行环境

项目已经自带环境初始化脚本：

- [deploy/linux/setup-ubuntu.sh](</C:/Users/Administrator/Desktop/word/sop-company-app/deploy/linux/setup-ubuntu.sh>)

上传后在服务器里执行：

```bash
cd /root/upload/sop-company-app
chmod +x deploy/linux/setup-ubuntu.sh
sudo bash deploy/linux/setup-ubuntu.sh
```

这一步会安装：

- `curl`
- `nginx`
- `rsync`
- `Node.js 20`

安装完成后先确认版本：

```bash
node -v
nginx -v
```

## 五、正式安装应用

项目已经自带正式安装脚本：

- [deploy/linux/install.sh](</C:/Users/Administrator/Desktop/word/sop-company-app/deploy/linux/install.sh>)

执行下面这组命令：

```bash
cd /root/upload/sop-company-app
chmod +x deploy/linux/install.sh
sudo APP_SLUG=sop-company-app \
APP_USER=flowdocs \
APP_DIR=/opt/sop-company-app \
DATA_ROOT=/var/lib/sop-company-app/data \
PORT=3100 \
HOST=127.0.0.1 \
DOMAIN=sop.yourcompany.com \
bash deploy/linux/install.sh
```

说明：

- `DOMAIN` 请替换成你自己的正式域名
- 如果你暂时还没有域名，也可以先把 `DOMAIN` 写成服务器公网 IP

这一步会自动完成：

1. 创建系统用户 `flowdocs`
2. 把程序安装到 `/opt/sop-company-app`
3. 把数据目录建立到 `/var/lib/sop-company-app/data`
4. 生成 `systemd` 服务
5. 生成 `Nginx` 反向代理配置
6. 启动应用服务

## 六、检查服务是否启动成功

执行：

```bash
systemctl status sop-company-app.service --no-pager
```

如果看到 `active (running)`，说明服务已经起来了。

再检查本机健康接口：

```bash
curl http://127.0.0.1:3100/api/health
```

正常时会返回类似：

```json
{
  "ok": true,
  "service": "Flow Docs"
}
```

再检查 Nginx：

```bash
nginx -t
systemctl status nginx --no-pager
```

## 七、配置 HTTPS

正式给公司使用时，强烈建议上 HTTPS。

先安装证书工具：

```bash
sudo apt update
sudo apt install -y certbot python3-certbot-nginx
```

然后申请证书：

```bash
sudo certbot --nginx -d sop.yourcompany.com
```

完成后再把环境变量文件里的 Cookie 安全项打开：

```bash
sudo nano /etc/sop-company-app.env
```

把：

```bash
SESSION_COOKIE_SECURE=false
```

改成：

```bash
SESSION_COOKIE_SECURE=true
```

保存后重启服务：

```bash
sudo systemctl restart sop-company-app.service
```

## 十、正式上线后的使用流程

员工使用流程：

1. 用自己部门账号登录首页
2. 点击“新建文件”
3. 在编辑器中起草内容
4. 保存到服务器
5. 点击“提交审核”

管理员使用流程：

1. 用 `admin` 登录首页
2. 在“待审核队列”查看提交文件
3. 打开文档
4. 在编辑器顶部工具栏里选择“审核通过”或“退回修改”

打印流程：

1. 打开文档进入编辑页
2. 点击顶部工具栏里的“文件”
3. 点击“打印预览”
4. 在预览窗口里打印或导出 PDF

## 十四、如果访问失败，按这个顺序排查

1. 先看应用是否运行

```bash
systemctl status sop-company-app.service --no-pager
```

2. 再看端口是否监听

```bash
ss -lntp | grep 3100
```

3. 再看 Nginx 是否正常

```bash
nginx -t
systemctl status nginx --no-pager
```

4. 再看应用日志

```bash
journalctl -u sop-company-app.service -n 200 --no-pager
```

5. 再看服务器防火墙和云安全组是否放行 `80` 和 `443`

## 十五、正式部署前最后确认

上线前请确认这 6 件事：

1. 域名已经解析到服务器公网 IP
2. 云安全组放行了 `22`、`80`、`443`
3. `sop-company-app.service` 状态为运行中
4. `nginx -t` 没报错
5. 能正常登录 `admin`
6. 能用部门账号新建文件、提交审核、管理员审核通过、打印预览
