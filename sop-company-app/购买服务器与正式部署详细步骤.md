# 购买服务器与正式部署详细步骤

本文按“公司正式上线”来写，适合你现在这个项目。

推荐路线分两种：

- 方案 A：买中国内地云服务器，正式商用，访问更稳，但要做 ICP 备案。
- 方案 B：买中国香港云服务器，上线更快，不做 ICP 备案，但中国内地访问体验通常不如内地节点稳定。

如果你是公司正式使用，我建议优先选方案 A。

## 一、先决定部署路线

### 方案 A：中国内地服务器

适合：

- 公司员工主要都在中国内地
- 需要长期正式使用
- 希望访问更稳定
- 可以接受做备案

要求：

- 域名解析到中国内地服务器时，需要做 ICP 备案
- 如果开网站对外服务，备案后通常还需要按要求办理公安联网备案

### 方案 B：中国香港服务器

适合：

- 想先快速上线
- 暂时不想做备案
- 接受后续再迁移

要求：

- 不需要中国内地 ICP 备案
- 访问中国内地用户时延和稳定性通常不如中国内地节点

## 二、推荐购买配置

适用于你现在这套系统：

- CPU：`2 核`
- 内存：`4 GB`
- 系统盘：`50 GB SSD` 以上
- 带宽：`5 Mbps` 起步
- 系统：`Ubuntu 24.04 LTS`

如果后面图片很多、人数变多，升级到：

- CPU：`4 核`
- 内存：`8 GB`

## 三、推荐买哪家

如果你的公司主要在中国内地办公，优先建议：

- 阿里云 ECS
- 腾讯云 CVM

## 四、阿里云购买步骤

### 第 1 步：注册账号并实名认证

先完成：

1. 注册阿里云账号
2. 完成企业实名认证
3. 开通云服务器 ECS

### 第 2 步：进入购买页面

进入阿里云 ECS 自定义购买页面后，按下面选：

1. 地域
   建议选离公司近的地域，比如华南、华东、华北。
2. 付费方式
   建议先选 `包年包月`。
3. 实例规格
   先选 `2 核 4G`。
4. 镜像
   选 `Ubuntu 24.04 LTS`。
5. 系统盘
   选 `ESSD` 或同级 SSD，`50 GB` 以上。
6. 公网
   必须开公网 IP。
7. 带宽
   建议 `5 Mbps` 起步。
8. 登录方式
   建议设置 root 密码，或绑定密钥。
9. 安全组
   创建一个新的安全组。
10. 主机名
   例如：`flowdocs-prod-01`

### 第 3 步：安全组规则

创建完成后，在安全组里至少放通这几个入方向端口：

- `22`，SSH 登录
- `80`，HTTP
- `443`，HTTPS

更安全的做法：

- `22` 端口只允许公司固定公网 IP 访问
- `80` 和 `443` 放开公网访问

## 五、腾讯云购买步骤

### 第 1 步：注册账号并实名认证

先完成：

1. 注册腾讯云账号
2. 完成企业实名认证
3. 开通 CVM

### 第 2 步：进入自定义配置购买

在腾讯云 CVM 自定义配置页按下面选：

1. 地域
   选离公司近的地域。
2. 机型
   选标准型即可。
3. 配置
   `2 核 4G`
4. 镜像
   `Ubuntu Server 24.04 LTS`
5. 系统盘
   `50 GB SSD` 以上
6. 公网带宽
   `5 Mbps` 起步
7. 登录方式
   设置密码或 SSH 密钥
8. 安全组
   新建安全组并放通 `22`、`80`、`443`

## 六、买完服务器后你手里要有这些信息

购买完成后，把下面信息记下来：

- 服务器公网 IP
- root 密码或 SSH 密钥
- 服务器地域
- 你的正式域名
- 域名管理平台账号

## 七、域名怎么处理

### 如果你还没有域名

去阿里云、腾讯云、DNSPod 或其他注册商买一个域名。

建议用公司自己的正式域名，例如：

- `sop.你的公司域名.com`
- `flow.你的公司域名.com`

### 如果你已经有域名

登录域名控制台，添加一条 A 记录：

- 主机记录：`sop`
- 记录类型：`A`
- 记录值：你的服务器公网 IP

这样访问地址就会变成：

- `https://sop.你的公司域名.com`

## 八、是否需要备案

### 你买的是中国内地服务器

要求：

- 如果域名要解析到中国内地服务器并对外提供 Web 服务，需要先做 ICP 备案

通常流程：

1. 买好中国内地云服务器
2. 买好域名
3. 到云厂商备案系统提交备案
4. 备案审核通过后再正式开放域名访问

### 你买的是中国香港或海外服务器

要求：

- 通常不需要中国内地 ICP 备案

## 九、连接服务器

买完后，在你本地 Windows 电脑里打开终端或 PowerShell，执行：

```powershell
ssh root@你的服务器公网IP
```

如果是第一次连接，会提示确认指纹，输入：

```powershell
yes
```

然后输入密码。

## 十二、正式安装系统

项目里已经有正式安装脚本：

- [deploy/linux/install.sh](</C:/Users/Administrator/Desktop/word/sop-company-app/deploy/linux/install.sh>)

执行命令：

```bash
cd /root/upload/sop-company-app
chmod +x deploy/linux/install.sh
sudo APP_SLUG=sop-company-app \
APP_USER=flowdocs \
APP_DIR=/opt/sop-company-app \
DATA_ROOT=/var/lib/sop-company-app/data \
PORT=3100 \
HOST=127.0.0.1 \
DOMAIN=sop.你的公司域名.com \
bash deploy/linux/install.sh
```

这一步会自动完成：

1. 创建服务用户
2. 复制项目到 `/opt/sop-company-app`
3. 建立数据目录 `/var/lib/sop-company-app/data`
4. 建立 systemd 服务
5. 建立 Nginx 配置
6. 启动服务

## 十三、检查服务是否正常

先看应用服务：

```bash
systemctl status sop-company-app.service --no-pager
```

再看健康接口：

```bash
curl http://127.0.0.1:3100/api/health
```

再看 Nginx：

```bash
nginx -t
systemctl status nginx --no-pager
```

## 十四、配置 HTTPS

正式使用一定建议上 HTTPS。

安装证书工具：

```bash
sudo apt update
sudo apt install -y certbot python3-certbot-nginx
```

签发证书：

```bash
sudo certbot --nginx -d sop.你的公司域名.com
```

配置成功后，修改环境变量：

```bash
sudo nano /etc/sop-company-app.env
```

把：

```bash
SESSION_COOKIE_SECURE=false
```

改成：

```bash
SESSION_COOKIE_SECURE=true
```

然后重启服务：

```bash
sudo systemctl restart sop-company-app.service
```
